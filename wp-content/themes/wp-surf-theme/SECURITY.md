# Security Policy

The WP SURF Theme is maintained as open source software.  
Security is taken seriously, but responsibility is shared between maintainers and implementers.

---

## Supported Versions

Only the **latest stable release** is supported for security updates.

Security fixes are applied to the most recent version only.  
Older versions are considered unsupported.

---

## Reporting a Vulnerability

If you discover a potential security vulnerability, please **do not** create a public GitHub issue.

Instead, report it privately via:

- A GitHub Security Advisory (preferred), or
- Direct contact with the project maintainers (see repository contact details)

Please include:

- A clear description of the issue
- Steps to reproduce
- Proof of concept if available
- Affected WordPress and PHP versions (if known)

Reports with sufficient detail can be assessed more efficiently.

---

## Disclosure Process

After receiving a report:

1. Receipt will be acknowledged when possible.
2. The issue will be assessed and validated.
3. If confirmed, a fix will be prepared.
4. A patched release will be published.

There is no guaranteed response time.  
This project is maintained in good faith and without a formal service level agreement.

---

## Scope

This policy applies only to:

- The WP SURF Theme codebase

It does not cover:

- Third-party plugins
- Custom modifications by implementers
- Hosting, server, or infrastructure issues
- WordPress core vulnerabilities
- Misconfiguration of installations

If the vulnerability originates outside the theme codebase, it falls outside this policy.

---

## Responsible Disclosure

We ask that reporters:

- Allow reasonable time for a fix before public disclosure
- Avoid publishing exploit details before a patch is released

Coordinated disclosure helps protect users.

---

## Final Note

Security is a shared responsibility.

If you identify a potential issue, report it responsibly so it can be addressed appropriately.
