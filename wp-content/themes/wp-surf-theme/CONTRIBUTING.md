# Contributing

Thank you for your interest in the WP SURF Theme.

This project is open source, but it is primarily maintained and developed by the SURF team. Contributions are welcome within the boundaries described below.

---

## Bug Reports

Bug reports are welcome.

Please include:

- Clear steps to reproduce
- Expected behavior
- Actual behavior
- WordPress version
- PHP version

Incomplete reports may not be investigated.

For security-related issues, please refer to `SECURITY.md`.

---

## Feature Requests

Feature requests may be submitted via GitHub Issues.

Please note:

- Not all requests will align with the project roadmap
- There is no guarantee that requested features will be implemented

The maintainers decide which features are accepted.

---

## Pull Requests

Pull requests are not automatically accepted.

If you would like to contribute code:

1. Open an issue first to discuss the proposed change
2. Wait for confirmation from the maintainers
3. Follow the existing architecture and coding standards

Large architectural changes or unsolicited refactors may be declined.

---

## Code Standards

Contributions must:

- Follow WordPress coding standards
- Avoid adding unnecessary external dependencies
- Keep business logic out of templates
- Maintain compatibility with supported WordPress and PHP versions

Clean, minimal and well-documented changes are preferred.

---

## Licensing

By submitting a contribution, you agree that your work will be licensed under the same GPL license as this project.

---

## Maintainer Rights

Project maintainers reserve the right to:

- Accept or decline any contribution
- Request changes before merging
- Close issues that fall outside the project scope

---

Thank you for helping improve the WP SURF Theme.
